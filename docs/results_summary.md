# Итоговые результаты

## Финальные конфигурации

| Задача | Конфигурация | TP / FP / FN | Precision | Recall | F1 |
| --- | --- | ---: | ---: | ---: | ---: |
| no_hardhat | proposed pf01 + `EventConsolidatorV3` | 13 / 0 / 3 | 1.000 | 0.813 | 0.897 |
| no_vest | old_vest_lowconf_fine + `EventConsolidatorV3` | 21 / 6 / 2 | 0.778 | 0.913 | 0.840 |

Оценка выполнена на пилотной ручной выборке. Подробное обоснование
параметров — в основном тексте ВКР.

## Кратко о сравнении с baseline

В процессе разработки система сравнивалась с несколькими подходами:

- покадровые detector-based решения (yolo10s / yolo11s)
  без подтверждения нарушения во времени;
- SH17 detector в исходной конфигурации;
- YOLO-World с zero-shot prompt-классификацией;
- CLIP / VLM baseline.

Базовые подходы давали либо избыточный шум на негативных видео, либо
пропускали короткие нарушения. Событийная логика с привязкой СИЗ к
человеку и консолидацией повторов уменьшала шум при сравнимом или
лучшем покрытии нарушений.

Сами baseline-скрипты в этот репозиторий не переносились — он содержит
только итоговое решение.

## Ограничения

- Метрики получены на пилотной выборке; для других доменов требуется
  повторная разметка.
- Качество зависит от ракурса камеры, освещения, разрешения и
  компрессии видео.
- Веса детекторов не входят в репозиторий.
- Для промышленного внедрения необходима валидация на целевом домене,
  нагрузочное тестирование и проверка устойчивости к атакам / шуму.
- Текущие результаты не являются промышленной сертификацией.
